## Visão Geral

A API de Eventos é um endpoint REST público para enviar eventos de rastreamento a partir de qualquer sistema backend — plataformas de e-commerce, CRMs, ferramentas de automação (n8n, Make, Zapier) ou aplicações personalizadas.

```
POST /v2/public/tracking/events
```

| Propriedade | Valor |
|-------------|-------|
| **Método** | POST |
| **Autenticação** | Nenhuma (endpoint público) |
| **Content-Type** | `application/json` |
| **URL Base** | `https://api.metrito.com` |

<Info>
A API é pública e não exige autenticação. O campo `domain` no payload identifica o contêiner de destino. Os eventos são processados de forma assíncrona — a API responde imediatamente após receber o evento.
</Info>

## Schema da Requisição

### Campos Obrigatórios

```json
{
  "domain": "sujaloja.com.br",
  "config": {
    "name": "NomeDoEvento"
  }
}
```

| Campo | Tipo | Descrição |
|-------|------|-----------|
| `domain` | string | Domínio associado ao seu contêiner de rastreamento (ex: `sujaloja.com.br`) ou o ID MTC (ex: `MTC-5X35GWQ`) |
| `config.name` | string | Nome do evento para dashboards e relatórios no Metrito |

### Referência dos Campos

<ResponseField name="domain" type="string" required>
  Domínio associado ao seu contêiner, ou o ID MTC (ex: `MTC-5X35GWQ`).
</ResponseField>

<ResponseField name="config" type="object" required>
  <Expandable title="config">
    <ResponseField name="name" type="string" required>
      Nome do evento para o Metrito. Pode ser qualquer string que faça sentido para você (ex: `Lead`, `Compra`, `FormularioEnviado`).
    </ResponseField>
    <ResponseField name="facebook" type="object">
      <Expandable title="facebook">
        <ResponseField name="name" type="string">
          Nome do evento enviado para a API de Conversões da Meta. Deve seguir os eventos padrão da Meta: `PageView`, `Lead`, `Purchase`, `AddToCart`, `InitiateCheckout`, `AddPaymentInfo`, `CompleteRegistration`.
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="eventId" type="string">
  ID único do evento. Gerado automaticamente como `{timestamp}_{aleatorio}` se omitido (ex: `1758388123_kk78h7u5sf`).
</ResponseField>

<ResponseField name="eventTime" type="number">
  Timestamp Unix do evento. Usa o horário atual se omitido.
</ResponseField>

<ResponseField name="data" type="object">
  <Expandable title="data">
    <ResponseField name="value" type="number">
      Valor monetário do evento (ex: `299.90`).
    </ResponseField>
    <ResponseField name="currency" type="string">
      Código de moeda ISO 4217 (ex: `BRL`, `USD`).
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="lead" type="object">
  <Expandable title="lead">
    <ResponseField name="id" type="string">
      ID explícito do lead. Gerado automaticamente como `lead_{AAAAMMDD}_{aleatorio}` se omitido quando dados de lead estão presentes.
    </ResponseField>
    <ResponseField name="name" type="string">Nome completo do lead.</ResponseField>
    <ResponseField name="email" type="string">Endereço de e-mail do lead.</ResponseField>
    <ResponseField name="phone" type="string">Telefone no formato E.164 (ex: `+5511999999999`).</ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="utm" type="object">
  <Expandable title="utm">
    <ResponseField name="utm_source" type="string">Fonte (ex: `facebook`, `google`)</ResponseField>
    <ResponseField name="utm_medium" type="string">Meio (ex: `cpc`, `email`)</ResponseField>
    <ResponseField name="utm_campaign" type="string">Campanha (suporta formato `nome|id`)</ResponseField>
    <ResponseField name="utm_content" type="string">Conteúdo (suporta formato `nome|id`)</ResponseField>
    <ResponseField name="utm_term" type="string">Termo ou posicionamento.</ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="meta" type="object">
  <Expandable title="meta">
    <ResponseField name="url" type="string">URL da página onde o evento ocorreu</ResponseField>
    <ResponseField name="title" type="string">Título da página</ResponseField>
    <ResponseField name="language" type="string">Idioma da página</ResponseField>
    <ResponseField name="referrer" type="string">URL de referência</ResponseField>
  </Expandable>
  Preenchido automaticamente quando o evento é enviado pelo navegador. Quando enviado pelo servidor, inclua esses dados explicitamente se disponíveis.
</ResponseField>

<ResponseField name="cookies" type="object">
  Cookies do navegador para correspondência nas plataformas de anúncios. Chaves mais comuns: `_fbp` (Meta), `_ga` (Google Analytics), `_fbc` (clique Meta).
</ResponseField>

## Resposta

### Sucesso (200 OK)

```json
{
  "success": true,
  "eventId": "1758388123_kk78h7u5sf",
  "leadId": "lead_20250920_xyz789",
  "timestamp": 1758388123
}
```

### Erro de Validação (400)

```json
{
  "message": "Domain is required",
  "errors": [
    { "field": "domain", "message": "Domain field is required" }
  ]
}
```

## config.name vs config.facebook.name

Esses campos têm finalidades distintas e ambos devem ser incluídos quando você quer encaminhar eventos para a Meta:

| Campo | Utilizado por | Finalidade |
|-------|---------------|------------|
| `config.name` | **Metrito** | Nome interno do evento para dashboards. Use qualquer nome que quiser. |
| `config.facebook.name` | **API de Conversões da Meta** | Deve corresponder a um evento padrão da Meta. |

<Warning>
Se `config.facebook` for omitido, o evento é registrado no Metrito mas **não** é enviado para a Meta. Inclua-o sempre que quiser atribuição no Meta Ads.
</Warning>

## IP e Geolocalização

| Enviado de | Detecção de IP | Geolocalização |
|------------|---------------|----------------|
| **Navegador** (frontend) | Automática via cabeçalhos da requisição | Automática pelo IP do usuário |
| **Servidor** (backend) | IP do servidor | Localização do servidor |

Ao enviar eventos de um backend, a geolocalização refletirá a localização do seu servidor, não a do usuário. Se disponível, inclua os dados de localização do usuário explicitamente no objeto `meta`.
